第一部分：原理概述

什么是置换检验（Permutation Test）

置换检验，是Fisher提出的一种基于大量计算（computationally intensive），利用样本数据的随机排列（置换检验的核心思想，故名Permutation test），进行统计推断的方法。因其对总体分布自由，特别适合用于总体分布未知的小样本数据，以及一些常规方法难以使用的假设检验情况。

置换检验的主要步骤

  1. 提出零假设，我们提出的零假设就是两组（或多组）样本没有差异或者说差异不显著。
  2. 计算统计量，比如两组的均值之差，记作t0。
  3. 随机生成置换样本，并计算置换样本的统计量。
  4. 重复第三步，进行多次置换，得到多个统计量ti，i=1,2,3,….。
  5. 计算P值，即计算大于t0的ti个数在所有统计量个数中的占比。
  6. 判断接受或拒绝零假设，通常我们取0.05作为阈值，若P值大于0.05则接收零假设，即认为样本没有差异或者说差异不显著；若P值小于0.05则拒绝零假设，即认为样本差异显著。

第二部分：代码实现

代码文件说明

在次级目录project下存放了五个R语言代码文本文件。它们分别是：

  1. main.R：
    本次实验的入口文件，负责控制整个实验的流程与代码文件调用。
    • 文件中的主函数main只接收一个参数：当前工作目录的绝对路径，代码内的所有路径是依据该绝对路径来计算相对路径，从而避免了移动目录后需要多次修改绝对路径。
    • 同时，可以在主函数main修改部分实验参数，从而快速地进行对比实验。
  2. sim_cna.R：
    用于生成模拟数据，并在模拟数据中插入基因变异位点与随机噪声干扰。
  3. CBS_data.R：
    用于对模拟的数据进行去噪。
  4. permutation_test.R：
    进行置换检验。
  5. calculation.R：
    计算敏感度和准确度的标准差与均值。

由于代码文件中的注释比较丰富，具体实现细节这里就不再多做赘述。
